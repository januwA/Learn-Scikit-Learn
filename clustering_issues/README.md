# GitHub Issues 聚类分析 (无监督学习)

本项目演示了如何使用 Scikit-Learn 的进行 **无监督学习 (Unsupervised Learning)**，通过 **Clustering (聚类)** 算法对 GitHub Issues 进行自动模式识别和归类。

## 什么是聚类 (Clustering)？

聚类是机器学习中处理“无标签数据”的核心技术。
- **目标**：将相似的数据点归为同一个簇 (Cluster)。
- **特点**：不需要预先告知机器什么是“Bug”或“功能建议”，机器通过计算文本之间的相似度，自动发现“物以类聚”的模式。

## 形象类比：整理扣子

- **分类 (Classification)**：你给孩子一堆写好名字（猫/狗）的识字卡片。孩子学习的是**规则**。
- **聚类 (Clustering)**：你给孩子一堆各种颜色的扣子，不说话。孩子观察后，自发把红色的放一堆，绿色的放一堆，大的放一堆，小的放一堆。孩子发现的是**模式**。

## 典型案例：电商精准营销

假设你是电商老板，有 100 万个用户的消费历史数据（月消费额、登录频率、领券数）：
1. **聚类运行**：机器将用户分成了 `簇 0`, `簇 1`, `簇 2`。
2. **观察中心点**：通过分析每个簇的平均数据：
   - `簇 0`：消费高、经常登录 → **“金主爸爸”**。
   - `簇 1`：消费低、领券多 → **“羊毛党”**。
   - `簇 2`：登录少、不消费 → **“路人甲”**。
3. **精准营销**：给“金主爸爸”推新品，给“羊毛党”发大额券，从而实现利润最大化。

## 核心流程

1. **数据获取**：使用 GitHub CLI 抓取原始 Issue 数据（JSON 格式）。
2. **文本分词**：使用 `jieba` 对中文标题和内容进行分词。
3. **特征提取**：通过 `TfidfVectorizer` 将文本转化为数学向量（TF-IDF）。
4. **运行算法**：执行 `KMeans` 算法，将 Issue 自动切分为 N 个簇。
5. **结果解读**：通过提取每个簇中的“关键词暗示”，由人工总结该簇的主题语义。

## 如何运行

### 1. 准备数据
确保已安装 `gh` CLI 并登录，运行以下命令获取数据：
```bash
gh issue list --repo kevinanew/game_portal --limit 500 --state all --json number,title,body > clustering_issues/issues.json
```

### 2. 执行聚类
在项目根目录下运行：
```bash
uv run python clustering_issues/issue_clustering.py
```

## 学习笔记：聚类 vs 分类

| 维度 | 分类 (Classification) | 聚类 (Clustering) |
| :--- | :--- | :--- |
| **学习类型** | 监督学习 | 无监督学习 |
| **输入数据** | 必须有标签（如：这是猫） | 只有原始数据 |
| **输出结果** | 预定义的语义标签 (Cat/Dog) | 机器生成的簇 ID (0, 1, 2...) |
| **核心逻辑** | 学习已有的模式 | 探索未知的模式 |

## 结果分析建议

聚类生成的 **簇 ID** 仅仅是数字，要理解其背后的含义，需要观察：
- **关键词暗示**：该簇中权重最高的词汇（如出现“对接、接口”，则该簇可能关于 API）。
- **中心点分析**：查看该簇中最具代表性的案例。
- **人工映射**：根据发现的规律，将 ID 映射为人类可理解的业务标签。
