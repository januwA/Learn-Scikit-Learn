## 核心笔记：Scikit-Learn 与 深度学习 (YOLO) 的区别

### 1. 为什么在图像识别上，YOLO “碾压”了 sklearn？
*   **特征提取能力**：
    *   **sklearn**：需要手动指定特征。比如本项目中将图片展平为像素，这种方式对物体的旋转、光照和背景极其敏感。
    *   **YOLO (深度学习)**：自动提取特征。模拟人类视觉，自动识别图像中的边缘、纹理和形状，具有极强的泛化能力。
*   **任务复杂度**：
    *   **sklearn**：擅长“分类”（这是猫还是狗）。
    *   **YOLO**：擅长“检测”（猫在哪里，有几只，坐标是什么）。

### 2. 那么 sklearn 还有用吗？
**绝对有用！** sklearn 是机器学习界的“瑞士军刀”，它的霸主地位在 **结构化数据**（表格数据）领域依然无可动摇：
*   **核心地盘**：金融风控、预测房价、推荐系统、医疗诊断等数值型数据任务。
*   **优势**：计算开销极低（CPU 即可运行）、模型可解释性极强、在小样本数据上表现优于深度学习。
*   **行业标准**：即使在使用 YOLO 时，数据的前期清洗、标准化（Scaling）和特征选择依然经常调用 sklearn。

### 3. 我在这一阶段学到了什么？
即使识别精度受限，但在 sklearn 阶段的练习让我掌握了 **AI 开发的标准流水线**：
1.  **数据获取与预处理**：从 API 下载 -> 灰度处理 -> 归一化。
2.  **标注逻辑**：理解了分类任务中“文件夹结构即标注”的概念。
3.  **模型驱动**：`fit` (训练) -> `predict` (预测) -> `predict_proba` (置信度)。
4.  **评估体系**：学会看 `Accuracy` (准确率)、`Precision` (精确率) 和 `Recall` (召回率)。

## 进阶笔记：机器学习 vs 深度学习

### 1. 它们的关系：包含而非对立
*   **人工智能 (AI)**：机器表现出的人类智能。
*   **机器学习 (ML)**：AI 的子集，侧重于让机器从数据中学习规律（Scikit-Learn 所在领域）。
*   **深度学习 (DL)**：ML 的子集，侧重于利用深层神经网络（YOLO、GPT 所在领域）。

### 2. 核心区别对比

| 维度 | 传统机器学习 (Vanilla JS) | 深度学习 (React/Next.js) |
| :--- | :--- | :--- |
| **特征工程** | **靠人手工定义**（如像素展平、HOG） | **机器自动学习**（提取纹理、形状、耳朵） |
| **硬件依赖** | **CPU** 即可运行 | 强依赖 **GPU** 为主 |
| **数据需求** | 小样本即可工作 | 需要大规模数据支撑 |
| **可解释性** | **强**（可解释为什么这么分类） | **弱**（黑盒模型，参数难以直观理解） |

### 3. 给开发者的建议
传统机器学习（sklearn）是深度学习的**逻辑基石**。虽然深度学习在视觉领域更强大，但传统机器学习在处理结构化数据（表格、业务指标）时更加高效、稳健且经济。
